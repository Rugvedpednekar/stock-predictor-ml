How StockPredictor ML Works

This project is a simple end-to-end machine learning web application. It connects a stock data source, a time-series prediction model, and a minimal front-end interface to forecast the next day’s closing price.

1. Data Ingestion

For a given ticker symbol, the backend downloads recent daily OHLC (Open, High, Low, Close) data. Only the closing price series is used as the target variable for prediction.

2. Feature Engineering

The raw price series is transformed into input features. For example, we can create lag features (previous n days’ closing prices), moving averages, or returns that help the model learn patterns.

3. Model Training

A regression model (e.g., Linear Regression, Random Forest Regressor, or LSTM) is trained on historical data. The model learns to map the engineered features to the next day’s close.

4. Prediction Endpoint

The backend exposes a REST endpoint, for example: /predict?symbol=AAPL. When the front-end calls this endpoint, the server:

  • Fetches the latest stock data.
  • Builds the input features.
  • Uses the trained model to predict the next closing price.
  • Returns JSON with the current price, predicted price, signal, and recent history.

5. Front-End Visualization

The Search page uses JavaScript to:

  • Send the ticker symbol to the /predict endpoint.
  • Plot the last 60 days of closing prices on a custom canvas-based line chart.
  • Display the predicted price, approximate confidence score, and a Buy/Hold/Sell signal.

Model Limitations

  • Uses only historical price data (no fundamental or news data).
  • Short training window can lead to overfitting or unstable predictions.
  • Markets are noisy and affected by many external factors the model cannot see.

Educational Use Only

This web app is developed as a coursework project to demonstrate how to integrate a machine learning model into a web application. The predictions are not intended for real trading, and nothing here should be interpreted as financial advice.
